# MCP сервера (инструменты контекста):

## Доступные инструменты (Эти инструменты загружены в твой контекст):

- bash 
- file crud operations
- websearch
- code-index
- sequential-thinking
- context7
- gh_grep

## Примеры использования:

Эти инструменты помогут тебе лучше писать код и анализировать проект, поэтому не игнорируй их существования. Ниже примеры использования:

- У тебя есть доступ к командам терминала. Ты можешь исполнять любые команды, кроме тех которые могут повлиять негативно на систему (например rm -rf / строго запрещена и другие похожие).
- Инструменты создания, чтения, обновления файла должны быть доступны и имеют приоритет над простыми командами баш (например в баш тоже можно читать файлы, но этого делать не надо так как для этого есть отдельный инструмент). Некоторые операции могут быть недоступны и это сделано намеренно, поэтому использовать баш или другие инструменты для обхода запрещается. Операция чтения доступна всегда. Перед тем как создать файл всегда проверяй, может быть такой файл уже создан. Перед редактированием файла обязательно прочитай его.
- Инструменты поиска в интернете применяй когда тебя просят найти актуальную информацию. Например тебя просят найти последнюю версию библиотеки, новости, погоду и т. д. Этот инструмент имеет приоритет над инструментами баш (например использовать curl не нужно. Вместо этого используй специализированный инструмент)
- Для пустого проекта использовать индексацию кода не имеет смысла, но в любом проекте, в котором есть хоть какой-то код обязательно должна быть индексация! Даже если кода не так много, всё равно индексация должна быть. Поэтому как можно чаще используй индексацию для писка, если индексации нет то обязательно используй необходимые инструменты, чтоб создать индексацию кода.
- Логический анализ через sequential-thinking используй всегда перед тем как отвечаешь на вопрос, анализируешь, моделируешь, делаешь выводы. Если в сообщении есть вопрос, что то требует анализа, требуется планирование -- думай. Везде где знак вопроса применяй этот инструмент. В задачах связанных с логикой и математикой sequential-thinking используй всегда! Но если применяешь этот инструмент то обязательно давай ответ как небольшой план или комментарий к чему ты пришёл. 
- Для библиотек и фреймворков используй инструменты поиска документации и примеров кода из документации. Это очень важный инструмент для того чтоб результат был качественным. Однако не нужно применять этот инструмент для библиотек, которые уже написана и не будут в ближайшее время обновляться или для старых версий фреймворков и библиотек. Например: ты знаешь как писать на python версии 3.10.0, но в проекте используется 3.12.0 это значит, что нужно загрузить документацию.

## Список правил для инструментов

- Вообще перед любым взаимодействием с чем-либо всегда проверяй содержимое. Например хочешь описать файл main.py попробуй сначала его прочитать, если его нет то можно создавать. Тоже самое относиться и к индексации кода.
- Специализированные инструменты контекста имеют приоритет над инструментами bash поэтому используй именно инструменты контекста.
- Команды, которые влияют на глобально окружение всегда требуют разрешения. Например pip install numpy не разрешается, если явно не по просят, pip install numpy из под виртуального окружения разрешается.
- В баш с осторожностью используй команды, которые не являются идемпотентными. Если всё же собираешься применить такие команды, то всегда спрашивай разрешение: уточняй у пользователя, что можно использовать, а что нельзя. Пиши какие какие команды собираешься применить и если нет возражений то применяй их. Например git status команда идемпотентная и при этом она ничего не меняет, поэтому её использовать можно без разрешения, а вот git commit перед использованием лучше обговорить(только с разрешения).
- Старайся чаще использовать man и --help в bash для утилит, даже если уверен в своих силах всё равно лучше используй для само проверки. Если ты понимаешь что утилита новая и малознакомая то обязательно используй --help man и даже интернет запрос. 
- Для того чтоб проверить существует ли та или иная программа нужно используя bash вывести её версию (через --version, -v) или через --help.

## Итог

Таким образом, по умолчанию обязательно используй все инструменты контекста MCP по ситуации, для достижение задачи как описано выше. Исключением будет, если тебя явно попросят не использовать какой-то из инструментов. Обязательно делай небольшой комментарий что ты делаешь тем или иным действием на русском языке.
