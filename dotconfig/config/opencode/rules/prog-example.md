# Пример цепочки рассуждения в программирование программирования:

## Пример №1:

### Задача:

Пользователь попросил написать калькулятор на языке питон.

### Цепочка мыслей:

Хочу уточнить у пользователя какой калькулятор нужен ? Для этого задам вопросы:

- Какой формат интерфейса ? (GUI/CLI/другой)
- Есть ли список операций, которые надо реализовать ? (Нужно реализовать какой-то строгий список операций или нужно реализовать все операции, которые есть в обычном калькуляторе)

> Пользователь сказал что ему нужен CLI калькулятор. Список операций: '+', '-', '/', '*'. Вот пример работы калькулятора который хотел бы видеть пользователь: calc 3 * -5 / (0.1 + 0.2)

Напишу план для пользователя, что я собираюсь делать:

План решения задачи:
- Подготовлю проект
  - Проверю есть ли гит, если нет инициализирую гит
  - Если нет.gitignore создам его
  - Добавлю в .gitignore список необходимых файлов
  - Создам ./src если ещё не создан
  - Создам ./tests если ещё не создан
  - Если в текущей папке уже есть файлы, которые должны лежать в ранее созданных папках, то перемещу их туда.
- Инициализирую проект, создав окружение (использую uv так как стандарт)
- Организую точку входа в приложение:
  - Создам ./src/main.py и реализую файл
- Тестовый запуск:
  - Запущу проект для проверки
  - Запущу форматер кода (ruff + black)
  - Запущу линтер (ruff)
- [Если есть ошибки то исправляем их а только потом переходим к следующему пункту]
- Реализовываю модуль для математических операций:
  - Создам отдельный модуль calc (это папка с \_\_init\_\_.py)
  - Реализуй в модуле файл с набором операций из списка.
  - Сделаем тесты в ./tests для каждого файла.
- Тестовый запуск:
  - Запущу проект для проверки
  - Запущу форматер кода (ruff + black)
  - Запущу линтер (ruff)
- [Если есть ошибки то исправляем их а только потом переходим к следующему пункту]
