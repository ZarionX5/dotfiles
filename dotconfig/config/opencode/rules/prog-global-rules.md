# Правила программирования

## Список правила программирования:

- Код должен быть чистым!!!
- Применяй принципы KISS, DRY и т. д.
- Декомпозируй всё: задачи, код, модули (Модули должны выполнять что-то одно но хорошо. Как в философии UNIX)
- Придерживайся стандартов индустрии программирования
- Используй инкрементную методологию разработки 
- Пиши понемногу, но много раз (нельзя писать кучу кода за раз. Пиши чуть-чуть и проверяй работоспособность этого кода)
- Используй линтеры и форматеры если это возможно (после того как код написан отформатируй его и проверь при помощи линтера) 
- Пиши код без комментариев (исключение: если явно просят написать комментарии в коде или просят само документируемый код)
- Предпочтение отдавай языкам со строй типизацией где это возможно (например используй по умолчанию ts вместо js, tsx вместо jsx и т. д.)
- Типизируй код где это возможно (например: в python нужно использовать модуль typing для типизации. В js нельзя указывать тип но можно использовать JSDoc для указания типов, но лучше уточнить можно ли использовать JSDoc)
- Обязательно сверяйся с документацией (ищи документацию через инструменты контекста, если таковые доступны)
- Задавай уточняющие вопросы (Если в вопросе есть противоречия, непонятные формулировки или появились сомнения лучше переспросить)
