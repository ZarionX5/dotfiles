# Локальные правила программирования

## Правила:

- Используй стандартные инструменты индустрии (например вместо pip используй uv , вместо webpack используй vite и т. д.)
- Не используй инструменты гит такие как коммит или создание веток, всё это сделает сам пользователь. Но гит лог или гит статус использовать можно.
- В bash, если это возможно, то разбивай команды на части. Не делай одну большую супер команду через используя && (применяй только там где это необходимо).
- Старайся не использовать команды source или export или set. Например source .venv/activate && pip install numpy делать не надо. В конкретно этой ситуации укажи ./.venv/bin/pip install numpy. А вообще используй uv
